WhaTapのURLモニタリングを適用するときに発生する可能性のあるエラータイプとそのタイプによる設定方法について説明します。

URLモニタリング画面に表示されるチャート領域をドラッグすると、その時点でURLを呼び出したときに取得した結果データが画面下部に表示します。データの詳細原因に表示されたメッセージによる対処方法を案内します。

<ImgLang img='url-error-type.png' desc='URLモニタリングエラータイプ' />

## Invalid URL

誤ったURLを入力する場合に発生するメッセージです。 

## Unknown Host or URL

このホストまたはURLが見つからない場合に発生するメッセージです。プロトコルを正しく入力しているか、正しいURL形式およびホストアドレスを入力しているかを確認してください。 

## Connect Time Out

誤ったURLに対する接続時間が超過して接続できないときに発生するメッセージです。サーバーへの接続の試行中に、最大所要時間の5秒以内に接続に失敗した場合に発生します。 

## Read Time Out

読み取り操作が指定された時間内に完了していないときに発生するメッセージです。サーバーからのデータ受信中に最大所要時間の10秒以内にレスポンスがない場合に発生します。

## Unexpected end of file from server

エージェントとサーバーのエンコード値が異なる場合、またはパラメータに対するエンコード方法が異なる場合に発生するメッセージです。 

## Connection Refused

URL呼び出しのための接続自体がブロックされた場合に発生するメッセージです。エージェントから呼び出しが不可能な場合で、該当URL、またはサーバーのファイアウォールの確認が必要です。

- ウェブブラウザでURLを入力して接続に問題がないかチェックしてください。入力中にエラーが発生する場合があります。

- ファイアウォールおよび頻繁な呼び出しに対するIPのブロック履歴を確認し、モニタリングエージェントのIPに対するアクセスを許可してください。 

## No route to host

目的地に到達するためのパスが存在しない場合に発生するメッセージです。DNSから情報が見つからない場合と、そのURLに関する情報がない場合、またはアップデートされていない場合に発生します。次の項目を正確に入力したことを確認してください。

- ポート指定

- 入力URLのプロトコル

  > 例、'htts://'と'https://'を間違って入力した場合
