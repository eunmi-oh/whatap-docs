May 30, 2024

- <Status>Feature</Status> Added the websocket package monitoring to collect data while transmitting data from the server to clients.

- <Status>Feature</Status> Added the agent setting options.

  - **ignore_nextjs_build_file_enabled** <span class='type'>Boolean</span>

    Default `true`

    Set whether to monitor the _Next.js_ build files. If this option is set to `true`, the _Next.js_ build files are not monitored.

  - **ignore_nextjs_build_file_path** <span class='type'>String</span>

    Default `/next/`

    You can specify the startup path of _Next.js_ build files to exclude from the monitoring. It is applied when the value of `ignore_nextjs_build_file_enabled` is `true`. With comma (,) as the delimiter, multiple paths can be set.

    ```ini title="whatap.conf"
    ignore_nextjs_build_file_path=/next/abc/,/next/def/
    ```

- <Status>Changed</Status> Changed the default value of the `trace_sampling_enabled` option to `true`.

- <Status>Fixed</Status> Modified the HTTP monitoring to ignore statistical data collection for socket errors.

- <Status>Fixed</Status> Fixed the issue where the URL is not displayed when the transaction result is <b>Lost Connection</b>.
