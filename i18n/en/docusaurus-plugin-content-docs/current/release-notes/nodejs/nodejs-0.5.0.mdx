August 12, 2024

- <Status>New</Status> Added the agent option to set the path of WhaTap agent log file.

  > **log_root** <span class='type'>String</span> Default none

  - The *logs* folder is created under the path set in the `log_root` option, and the WhaTap log file is created in that folder.

  - If you set an agent option as in the following example, the path for creating the whatap log file is as follows:

    > <em>/Users/sample_user/workspace/nodejs_application/whatap_logs/logs/</em>

    ```ini title="whatap.conf"
    log_root=/Users/sample_user/workspace/nodejs_application/whatap_logs/
    ```

- <Status priority='1'>Feature</Status> Added the monitoring feature for the Node.js's built-in function, `fetch`.

- <Status priority='1'>Feature</Status> Added the Redis monitoring package ([ioredis](https://www.npmjs.com/package/ioredis)).

- <Status priority='1'>Changed</Status> Modified the name of the option to ignore the build file monitoring.

  - `ignore_nextjs_build_file_enabled` → `ignore_build_file_enabled`
  - `ignore_nextjs_build_file_path` → `ignore_build_file_path`

- <Status priority='0'>Fixed</Status> Fixed the error where the value of the `auto_oname_prefix` option is applied as `true` even though it is `false`.
